
# Android UI Structure Analyzer

English documentation: [README.md](README.md)

AndroidスタイルのUI XML構造を解析し、UI要素の情報を抽出するPython製の技術検証プロジェクトです。

## 概要

本ツールは、resource-id を用いてUI要素を検索し、bounds（座標範囲）情報を抽出して中心座標を計算します。
さらに、大学でのウェルネス研究（睡眠の質と幸福度）の知見を応用し、機械的な同一座標クリックを防ぐ「人間らしいタップ位置のゆらぎ」を動的にシミュレートする機能を搭載しています。

Android UIレイアウトの構造理解や、人間工学・認知科学を融合したUI自動化技術の学習を目的として開発しました。

## 主な機能

* XMLレイアウトの高速解析（正規表現による頑丈なパース）
* resource-idによるUI要素の自動検索
* bounds（領域座標）情報の抽出
* UI要素の幾何学的中心座標の計算
* **【独自機能】ボタンサイズに応じたタップ位置の動的「ゆらぎ」生成（human_tap_point）**

## 使用技術

* Python 3
* 正規表現（`re` モジュールによる最短一致パース）
* Dataclasses（モダンなデータ構造設計）
* 例外処理（検証用XML自動生成による堅牢な設計）

## 実行結果例

本リポジトリに同梱されている `sample_window_dump.xml` を読み込んで実行すると、以下のように機械的な中心座標と、そこから動的に計算された「人間らしいゆらぎ座標（Human-like Point）」がログ出力されます。

## このプロジェクトで学んだこと

* **学際的アプローチの具現化（技術×感性の融合）**
  大学（健康学部）で培った「一定の機械的リズムが与える負荷（ストレス）や認知特性」という知見を、エンジニアリングにおける「座標のゆらぎアルゴリズム（`human_tap_point`）」へ応用。自身のバックグラウンドを技術的ロジックへと昇華させる設計手法を学びました。
* **正規表現を用いた構造化データの高度な抽出**
  複雑なXML階層の中から、特定の属性（`resource-id`）とそれに紐づく座標データ（`bounds`）のみをピンポイントで切り出す、最短一致（`.+?`）を用いた堅牢なパターンマッチング技術を習得しました。
* **モダンなデータ構造設計とカプセル化**
  Pythonの `@dataclass` を用いたスマートなオブジェクト管理や、幾何学的中心やゆらぎ座標の計算を `@property` で隠蔽（カプセル化）する、可読性と保守性の高いオブジェクト指向的なコード設計を実践しました。
* **エラーハンドリングと他者への利用配慮**
  指定ファイルが存在しない場合でもシステムをクラッシュさせず、内部データによるデモ動作へ安全にフォールバックさせる例外処理を実装。成果物としての「頑丈さ」と「扱いやすさ」を意識した開発手法を学びました。
* **デバッグによる問題解決能力の向上**
  開発初期、対象のUI要素が正常に検出できない不具合に直面しました。XMLのパース対象を詳細にログ出力して調査した結果、Pythonの正規表現におけるドット（`.`）がデフォルトでは改行コードにマッチしないことが原因であると特定。フラグ引数に `re.DOTALL` を指定することで問題を根本解決し、正規表現の仕様への深い理解と、原因を論理的に追究するデバッグの重要性を学びました。

## 開発背景

本プロジェクトは、個人的に検証を重ねていた「Android実機におけるUI解析および自動化技術」の概念実証（PoC）をベースに、一般公開用としてコードを抽象化・クリーン化したものです。

元のプロトタイプでは、Android端末上のTermux環境およびUIAutomatorを活用し、実機の画面からリアルタイムにレイアウトXMLをダンプして座標を抽出、実際のタップ操作までを連携させるモバイル特化型のシステムとして構築していました。

今回の公開版にあたっては、閲覧者が特定のアプリや実機環境（ADB接続など）に依存することなく、PC上のPython環境単体でスムーズに「高速パース」と「ゆらぎ生成ロジック」の挙動を確認できるよう、検証用のサンプルXML（`sample_window_dump.xml`）をリポジトリ内に最初から同梱する構成へと最適化を行いました。
